# Contributing

## Issues: open

If you spot a factual error, a documentation gap, a contradiction between this repo and the visible live dashboard, or a question about how the methodology works — please open an issue. This is the primary surface for engagement with the methodology.

Useful issue shapes:

- **Correction** — "page X claims Y but actually Z"
- **Question** — "what does X mean?" / "why this choice over Y?"
- **Methodology challenge** — "have you considered X failure mode?"
- **Consistency** — "this file disagrees with that file"

## Pull requests: closed

This is a single-author project for the duration of the 1,095-day experiment. The methodology represents one operator's specific experimental setup; merging contributions would dilute that, and the credibility posture of the experiment depends on it being one specific bot rather than a community-tuned consensus.

If you have improvements you want to publish, fork freely — the documentation license is CC-BY-SA-4.0. Don't expect changes merged upstream.

## Methodology proposals during the experiment

No. Methodology changes during the experiment are operator decisions, logged in [meta/](meta/) when they happen, on a deliberately rare cadence. External proposals are welcome as issues for discussion but are not actioned mid-experiment. Day 1,095 is the right window for "this whole thing should have worked differently" — there's a retrospective for that.

## Code contributions

This repo contains documentation only. The trading bot's source code is closed (see [what-is-NOT-here.md](what-is-NOT-here.md)) and is not accepting outside contributions.

## Code of conduct

Be useful, be skeptical, be kind. Bad-faith engagement — dunking, harassment, low-effort drive-bys — will be locked without comment.
